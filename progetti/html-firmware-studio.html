Panoramica

HTML Firmware Studio è un’applicazione sviluppata per semplificare l’inserimento di pagine HTML nei firmware di Arduino, ESP32, ESP8266 e schede programmabili in Python. Il programma trasforma automaticamente il contenuto HTML nel formato richiesto dalla piattaforma selezionata, evitando la conversione manuale di virgolette, caratteri speciali e righe di codice.

Il funzionamento è bidirezionale: oltre a generare il codice da incorporare nel firmware, l’applicazione può analizzare codice Arduino, C++ o Python già esistente e ricostruire la pagina HTML originale. Questo permette di modificare interfacce web già integrate nei microcontrollori senza doverle riscrivere da zero.

Problema risolto

Inserire una pagina web direttamente in un firmware è un’operazione ripetitiva e soggetta a errori. Ogni riga può richiedere escape, concatenazioni, macro, stringhe raw oppure istruzioni specifiche del server utilizzato. HTML Firmware Studio centralizza queste conversioni in un’unica interfaccia e consente di cambiare profilo senza riscrivere manualmente il contenuto.

Profili di esportazione

  1. Arduino client.print / client.println: adatto a WiFiClient, EthernetClient e altri oggetti compatibili con Print.
  2. C++ raw string in PROGMEM: genera una variabile HTML residente nella memoria programma.
  3. ESP32 WebServer ed ESP8266WebServer: genera il codice per server.send_P con route e Content-Type configurabili.
  4. ESPAsyncWebServer: genera la risposta tramite request->send_P.
  5. MicroPython socket: crea PAGE_HTML e il relativo invio tramite socket.
  6. CircuitPython Adafruit HTTPServer: prepara la risposta con l’oggetto Response.

Conversione inversa automatica

Il motore riconosce automaticamente le istruzioni client.print e client.println, comprese quelle racchiuse nella macro F(), le stringhe C++ raw archiviate in PROGMEM e le assegnazioni Python prodotte dal programma. Il contenuto viene decodificato e riportato nell’editor HTML mantenendo, quando possibile, righe vuote e indentazione.

Flusso di utilizzo

  1. Si apre o si incolla la pagina HTML nell’editor sorgente.
  2. Si seleziona il profilo di destinazione.
  3. Si configurano nomi, route, Content-Type e opzioni del profilo.
  4. Il programma genera il codice firmware nell’editor di output.
  5. Il risultato può essere copiato oppure salvato nel formato corretto.
  6. Con lo scambio degli editor è possibile eseguire la procedura inversa.

Gestione dei file

L’applicazione apre file HTML e sorgenti firmware con estensioni .html, .htm, .ino, .cpp, .h, .hpp, .py e .txt. Sono disponibili comandi per aprire, copiare, incollare, scambiare i due editor e salvare l’output con un’estensione coerente al profilo selezionato.

Impostazioni avanzate

  • Nome personalizzabile del client, server o socket.
  • Nome configurabile della variabile contenente il codice HTML.
  • Route HTTP e Content-Type modificabili.
  • Macro F() opzionale per ridurre l’utilizzo della RAM.
  • Intestazioni HTTP opzionali per i profili che le richiedono.
  • Selettore rapido del profilo direttamente nel convertitore.

Interfaccia e documentazione

La GUI utilizza il tema Mannino Maker scuro con accenti verdi e può essere commutata anche in modalità chiara. La navigazione comprende il convertitore, un manuale interno completo e una pagina di impostazioni. Il manuale contiene esempi dedicati alle piattaforme supportate e permette di utilizzare il programma senza consultare documentazione esterna.

Qualità e test

Il progetto comprende test automatici per verificare la reversibilità delle conversioni. Sono inoltre inclusi esempi pronti per ESP32, ESPAsyncWebServer, Arduino Ethernet, MicroPython e CircuitPython, insieme agli script per preparare, avviare e compilare la versione Windows.

Versione 2.0.3

La versione 2.0.3 migliora il branding dell’applicazione, la visualizzazione del logo nei temi chiaro e scuro, i margini di sicurezza dell’icona e la generazione delle icone per Windows, Android e macOS.

Destinatari

Il software è pensato per maker, studenti, sviluppatori embedded e tecnici che realizzano pannelli di controllo, pagine di configurazione, dashboard locali e interfacce web ospitate direttamente su microcontrollori.
